# Plan

## Goal

Restore the old `REVIEW.md` format — the commit-hash marker and per-hunk
checkboxes — in the `REVIEW.md` that the **Clean** state generates, **without**
re-introducing any enforcement that gates the workflow on those markers/boxes.

## Findings (current state of the code)

- `REVIEW.md` is generated by the **Clean** state. The prompt is
  `src/prompts/clean.md`. It currently produces a _markerless_ format:
  `# Review`, `## <Chunk Title>`, plain bullet pointers `- ./path.ts#42` — no
  hash marker, no `- [ ]` checkboxes.
- The **old** format (commit `3570a1a`'s parent, prompt `human-review.md`) was:
  - heading `# Review: <short-hash>` (first 7 chars of base ref)
  - marker `<!-- base: <full-hash> -->` (full SHA of the review base)
  - per-hunk checkboxes `- [ ] ./path/to/file.ts#42`
- **Enforcement is already gone.** Commit `3570a1a` added a
  `reviewHasUncheckedBoxes` → `review-incomplete` gate, but the later `04f43ee`
  "16-state resolver" cutover dropped it. Today the machine (`src/Machine.ts`,
  review branch ~L425-449) routes purely on git facts:
  - `reviewCommitted` (committed + clean tree) → **Done** (`gtd: done`, removes
    REVIEW.md).
  - `reviewDirty` (committed + any pending edit) → **Accept Review** (seed new
    plan).
  - uncommitted REVIEW.md → **Await Review** (commit `gtd: awaiting review`). No
    code reads checkboxes or the base marker. So "not enforced any more" is
    already satisfied at the machine/edge level — `src/Events.ts` has no
    `UncheckedBoxes` / `[ ]` / base-marker logic.
- `reviewBase` (the full base hash) already exists on `ResolvePayload` and is
  passed through to `ResolveContext` (`src/Machine.ts` L179), but it is **not
  rendered into the prompt**. `src/Prompt.ts` (L197-199) only inlines `refDiff`
  for the clean state; the literal base hash never reaches the agent, so the
  agent cannot author `# Review: <short-hash>` / `<!-- base: <full-hash> -->`
  reliably.

## Changes

### 1. `src/prompts/clean.md` — restore the old output format

Rewrite the "Write `REVIEW.md`" format block to match the old `human-review.md`
format:

- heading `# Review: <short-hash>` (first 7 chars of the base hash)
- `<!-- base: <full-hash> -->` marker line
- per-hunk checkboxes: `- [ ] ./path/to/file.ts#42` Keep: semantic hunk
  grouping, relative `./` pointers, drift-hint line numbers, ≤6-word imperative
  titles, the `gtd format REVIEW.md` normalize step, leaving REVIEW.md
  **uncommitted**, and the existing re-run/approve/request-changes wrap-up text.
  Do **not** add any "you must check all boxes" language — the boxes are
  navigational aids for the human, not a gate.

### 2. `src/Prompt.ts` — surface the base hash to the clean prompt

The prompt needs the full + short base hash to fill the marker/heading.
Recommended: extend the clean-state block at L197-199 to inline the literal
`reviewBase` hash (label the diff `git diff <reviewBase> HEAD` with the literal
hash, or add a `Review base: <full-hash>` line), so the agent can read the hash
from context. `reviewBase` is already on `ResolveContext`, so no machine/edge
change is needed.

### 3. Tests

- `src/Prompt.test.ts`: add a case asserting the rendered **clean** prompt
  contains the literal `reviewBase` hash (so the agent can author the marker).
  Update any existing clean-prompt assertions for the new diff label.
- `tests/integration/features/review.feature`: existing scenarios use the
  markerless format only as _fixtures_ and assert routing (commit subject /
  REVIEW.md presence), which is format-agnostic — they stay green. Add a
  composable scenario proving a committed REVIEW.md that contains the
  `<!-- base: -->` marker **and** unchecked `- [ ]` boxes still routes to
  **Done** on a clean tree — i.e. unchecked boxes do **not** block approval.
  This is the regression guard for "not enforced any more".

### 4. README

Update `README.md` REVIEW.md description (L62 and the Clean row ~L235) to
mention the restored hash marker and hunk checkboxes, noting they are
informational and not enforced.

## Notes

- No `src/Machine.ts` or `src/Events.ts` changes are required — enforcement is
  already absent and the base hash already flows through to the context.
- `STATES.md` is the redesign target and intentionally diverges; per memory,
  defer STATES.md sync.
